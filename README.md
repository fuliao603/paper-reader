# Paper Reader

Paper Reader 是一个本地运行的 PDF 学术阅读与翻译桌面工具，基于 Vite、React、Electron 和 Express 构建。它面向阅读英文论文、教材章节、技术文档和截图型资料的场景，提供 PDF 阅读、滑词翻译、区域 OCR、图解翻译、对照翻译、批注高亮、笔记、翻译历史以及导入导出等功能。

项目默认将配置、最近打开记录、翻译历史、笔记和批注数据保存在 Electron 的 `userData` 目录中，不使用浏览器 `localStorage`。PDF 本体不会被复制到配置目录；当用户选择将高亮写入 PDF 本体时，程序会直接修改对应 PDF，并创建备份文件。

## 功能特性

### PDF 文献导入与阅读

- 支持导入本地 PDF 文献并在桌面端阅读。
- 支持上一页、下一页、页码输入跳转。
- 支持鼠标滚轮阅读：当前页可滚动时优先滚动当前页，只有到达顶部或底部后继续滚动才翻页。
- 支持 PDF 缩放、缩放比例输入、全屏阅读。
- 支持关闭当前 PDF，关闭后不删除最近打开记录、翻译历史、笔记和批注。
- 支持最近打开记录与阅读进度保留：
  - 文件路径；
  - 文件名；
  - 文件大小；
  - 最后打开时间；
  - 最后阅读页码；
  - PDF 总页数；
  - 当前缩放比例；
  - 右侧栏宽度；
  - 右侧栏显示状态。
- 支持从“最近打开”重新打开 PDF 并恢复阅读状态。

### 滑词翻译

- 支持在 PDF 文本层中划选英文内容后自动翻译。
- 支持保持公式、上下标、专业术语、缩写和变量表达。
- 翻译结果显示在右侧“翻译结果”模块。
- 右侧结果区提供复制和清空按钮。
- 翻页、缩放、全屏切换不会自动清空右侧结果，只有点击“清空”或关闭当前 PDF 才清空当前结果。

### 区域 OCR 与截图翻译

点击顶部工具栏的“区域 OCR”后，可以选择不同 OCR 模式，对 PDF 页面中的截图、图表、扫描文字或复杂区域进行翻译。

#### 文本模式

- 框选 PDF 区域后进行 OCR 识别和翻译。
- 右侧结果区显示：
  - 框选区域截图；
  - 可编辑 OCR 文本；
  - 重新翻译按钮；
  - 翻译结果。
- 用户可手动修正 OCR 文本后重新翻译。
- 重新翻译不会重新 OCR、不会重新截图、不会重新框选，只把修正后的文本发送给翻译接口。
- 文本模式截图可点击打开大图查看。

#### 图解模式

- 适合示意图、流程图、结构图、图表说明等需要结合图像理解的内容。
- 框选区域后生成带中文翻译覆盖的结果图片。
- 结果图片会弹窗显示，也会保留在右侧文本框中。
- 关闭弹窗后，可再次点击右侧结果图片重新打开。
- 图解结果弹窗支持缩放、拖动和全屏查看。

#### 对照模式

- 适合文字密集图片、扫描版段落或需要同时查看原图和译图的内容。
- 框选区域后生成原图和译文覆盖图。
- 支持对照弹窗查看，关闭后仍可从右侧结果入口重新打开。
- 对照弹窗支持缩放、拖动和全屏查看。
- 对长方形图片会根据图片比例自动选择更合适的上下对照或左右对照布局。

### 右侧栏模块

右侧栏包含三个模块：

- 翻译结果；
- 翻译历史；
- 笔记。

“翻译结果”模块用于显示当前最近一次结果；“翻译历史”和“笔记”模块只显示当前 PDF 对应的数据，不会混入其他 PDF。

右侧栏支持：

- 复制当前译文；
- 清空当前结果；
- 隐藏 / 恢复右侧栏；
- 拖动调整右侧栏宽度；
- 独立滚动，不触发 PDF 翻页。

### 翻译历史

- 普通滑词翻译、文本模式 OCR、图解模式 OCR、对照模式 OCR 成功后会自动保存到当前 PDF 的翻译历史。
- 每篇 PDF 独立保存翻译历史。
- 每篇 PDF 最多保留 50 条翻译历史。
- 最近 30 个打开过的 PDF 会分别保留自己的翻译历史。
- 点击历史记录可恢复当时的右侧结果，不会重新 OCR、不会重新翻译。
- 图解和对照历史恢复后，仍可重新打开原结果弹窗。
- 文本 OCR 历史恢复后，仍可编辑 OCR 文本并重新翻译。
- 支持单条删除、批量选择删除、清空当前文献翻译历史。
- 支持导入和导出当前 PDF 的翻译历史。

### 笔记

- 支持从翻译结果主动添加笔记，不会把所有历史自动变成笔记。
- 支持以下笔记来源：
  - 普通滑词翻译；
  - 文本模式 OCR；
  - 图解模式 OCR；
  - 对照模式 OCR；
  - 批注高亮。
- 每篇 PDF 独立保存笔记。
- 支持查看笔记列表、打开笔记详情、编辑笔记、删除笔记。
- 点击笔记可跳转到对应页。
- 删除 OCR 笔记时，对应 OCR 小便签标签会同步删除。
- 删除高亮批注笔记时，不删除高亮，只解除高亮和笔记的绑定。
- 支持单条删除、批量选择删除、清空当前文献笔记。
- 清空笔记只清空当前 PDF 的笔记，不清空翻译历史，不删除 PDF 文件。

### 批注高亮

- 顶部工具栏提供“批注”按钮。
- 支持多种荧光高亮颜色。
- 支持“空”状态：选择空图案后，划选文字不会生成新高亮。
- 支持边划选边显示临时高亮预览，松开鼠标后保存正式高亮。
- 高亮记录与当前 PDF 绑定。
- 点击未绑定笔记的高亮可添加批注笔记。
- 点击已绑定笔记的高亮可打开右侧笔记详情。
- 右键点击高亮可取消高亮。
- 支持将高亮写入 PDF 本体，写入后用其他 PDF 软件打开也能看到高亮。
- 写入 PDF 本体时不会把笔记正文写入 PDF annotation，也不会创建 sticky note / popup annotation。
- 已修复写入 PDF 本体的高亮再次用 Paper Reader 打开时，高亮上方出现重复文字的问题。
- 取消已写入 PDF 本体的高亮时，会尝试同步删除 PDF 文件中的对应高亮。

### OCR 小便签标签

- 从文本 OCR、图解 OCR、对照 OCR 结果添加笔记后，PDF 页面对应区域会显示小便签标签。
- 小便签标签只在 Paper Reader 内部显示，用于快速跳转到对应笔记。
- 点击小便签标签可打开右侧笔记详情。
- 支持隐藏 / 显示笔记标签。
- 删除对应 OCR 笔记后，小便签标签会同步消失。

### 导入与导出

右侧栏和设置页都提供导入导出能力。

#### 单篇导入导出

- 翻译历史模块支持导出 / 导入当前 PDF 的翻译历史。
- 笔记模块支持导出 / 导入当前 PDF 的笔记。
- 导出笔记时会同时导出相关 annotations，包括文字高亮和 OCR 小便签标签。
- 导入笔记后，右侧笔记、高亮、小便签标签会恢复。
- 导入采用合并去重，不覆盖已有数据。

#### 设置页“导入与导出”

- 默认导出位置：
  - 可选择默认导出目录；
  - 可恢复系统 Downloads 默认目录。
- 批量导入：
  - 可选择多个 `.paperreader.json` 文件；
  - 支持单篇文件、合并文件和完整备份文件；
  - 导入完成后显示导入数量和跳过重复数量。
- 批量自选导出：
  - 显示本地已保存数据的 PDF 列表；
  - 可勾选多个 PDF；
  - 可选择只导出翻译历史、只导出笔记、或完整备份；
  - 支持每篇一个文件的分开导出；
  - 支持多篇合并导出；
  - 选择“分开导出”时导出名称输入框禁用；
  - 选择“合并导出”时可填写导出名称。
- 合并导出文件：
  - 支持点击“+”逐个添加导出文件；
  - 支持移除待合并文件；
  - 点击“合并”时弹出命名框；
  - 合并文件名以用户命名为核心，已去除重复的“翻译历史_PaperReader”等冗余前缀。
- 拆分导出文件：
  - 可将合并导出文件拆成多个单篇文件；
  - 拆分时保留原始类型，翻译历史仍拆为翻译历史文件，笔记仍拆为笔记文件，完整备份仍拆为完整备份文件。

导出文件统一使用 `.paperreader.json` 扩展名，并包含 schemaVersion、appName、exportType、exportMode、exportName、document metadata 和对应数据。

### 设置

设置页包含三个区域，视觉风格已统一为 card 分组：

- 模型设置；
- 翻译设置；
- 导入与导出。

模型设置支持：

- DeepSeek；
- OpenRouter；
- OpenAI-compatible 自定义接口；
- 自定义 Base URL；
- 自定义 API Key；
- 自定义模型名称；
- 常用模型预设。

翻译设置支持：

- 自定义翻译 Prompt；
- 恢复默认 Prompt；
- 导入术语库；
- 查看术语库；
- 清空术语库。

术语库可使用 CSV 或 JSON 格式导入，翻译时会作为固定译法参考。

## 使用场景

Paper Reader 适合以下场景：

- 阅读英文论文、教材、课程讲义和技术文档；
- 对 PDF 中的段落进行快速滑词翻译；
- 对扫描版 PDF、截图、图表和流程图进行 OCR 翻译；
- 在阅读文献时保存翻译历史和精选笔记；
- 为重点内容做高亮批注，并在后续继续阅读时恢复；
- 将翻译历史、笔记和批注导出备份，或在不同环境之间迁移；
- 需要使用自定义 OpenAI-compatible 接口或第三方模型网关进行翻译。

## 安装与运行方法

### 环境要求

- Node.js；
- npm；
- Windows 桌面环境。

### 安装依赖

```bash
npm install
```

### 桌面端开发运行

```bash
npm run electron:dev
```

该命令会先执行 Vite 构建，再启动 Electron 桌面应用。

### 仅构建前端

```bash
npm run build
```

### 代码检查

```bash
npm run lint
```

### 打包 Windows 桌面版

```bash
npm run dist
```

打包结果会输出到 `release` 目录。项目配置了 Windows 安装包和便携版目标，并使用 `build/icon.ico` 作为应用图标。

### 浏览器开发模式

如需分别启动前端和后端，可使用：

```bash
start-backend.bat
start-frontend.bat
```

浏览器模式主要用于开发调试。桌面端功能依赖 Electron API，部分本地文件读写能力在普通浏览器环境中不可用或会降级。

## 数据保存位置

Paper Reader 优先使用 Electron `userData` 目录保存本地数据，包括：

- 模型与翻译配置；
- 术语库；
- 最近打开记录；
- 每篇 PDF 的翻译历史；
- 每篇 PDF 的笔记；
- 每篇 PDF 的 annotations；
- 默认导出目录设置。

这些数据不会自动上传，也不会发送给翻译接口。翻译接口只接收用户主动选择或 OCR 得到、需要翻译的文本。

## 注意事项

- 请妥善保管 API Key，不要提交到代码仓库，也不要写入 README、截图或公开说明中。
- `.env`、Electron userData 中的配置文件和导出文件可能包含个人使用信息，分享前请自行检查。
- 将高亮写入 PDF 本体会修改 PDF 文件。程序会创建备份文件，但仍建议在重要文献上先自行备份。
- Paper Reader 内部的小便签标签和笔记系统不会写入 PDF 本体；写入 PDF 本体的是标准高亮标记。
- 如果移动或删除原 PDF，最近打开记录可能失效，软件会提示文件不存在或已移动。
- 导入数据采用合并去重策略，不会默认覆盖已有翻译历史、笔记和批注。
- 模型名称、API Base URL 和可用模型会随服务商变化而变化，如接口不可用，请到对应平台确认最新配置。
